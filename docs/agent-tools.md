# 智能体工具

当前版本智能体是工具优先实现。工具调用和策略判断由后端确定性执行，当前大模型只在这些结果之后参与自然语言分析。

## 已实现工具

- `get_current_room_state`：返回当前模拟房间指标、健康分、异常和建议。
- `query_sensor_history`：返回二氧化碳等指标的聚合证据。
- `create_automation_rule`：只创建草案；保存必须通过用户确认。
- `control_device`：将设备动作请求送入策略引擎和审计日志。
- `policy_check`：记录提示注入或绕过策略的拒绝决定。
- `llm_response_generation`：可选语言生成层，读取当前模型配置，把工具结果整理为更自然的中文解释。

## 回复结构

`POST /api/agent/chat` 返回：

- `message`：智能体回复。
- `used_data`：回复使用的数据来源。
- `tool_calls`：结构化工具证据。
- `policy`：相关安全策略判断。
- `needs_confirmation`：下一步是否需要用户确认。
- `model_usage`：本次是否使用当前大模型、使用的厂商和模型、失败或阻止原因。
- `rule_draft`：必要时返回规则草案。

## 模型接入边界

`/models` 页面提供中国区模型厂商配置、连接测试和当前模型选择，已预置小米 MiMo 与 Kimi 接口。

模型接入遵守以下边界：

- 智能体先把用户问题映射到工具，再用工具返回结构化结果。
- 设备控制类意图必须先经过策略引擎。
- 策略拒绝、提示注入、未知插座、报警器关闭等请求不会外发给大模型。
- 模型调用失败、未配置密钥或返回空文本时，自动回退到本地工具链回复。
- 审计日志会记录工具列表和模型使用状态，但不会记录明文密钥。
