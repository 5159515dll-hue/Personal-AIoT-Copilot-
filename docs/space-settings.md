# 房间与空间设置

当前工程不再假设只有一个房间。`/spaces` 页面和 `/api/spaces` 接口用于管理多个空间，例如书房、卧室、客厅、实验台、阳台或其他区域。

## 当前能力

- 新增空间：设置名称、类型、位置标签、楼层、时区。
- 区域划分：例如书桌、床头、门口、沙发区、实验台。
- 设备绑定：保存该空间关联的 `device_id` 列表，后续设备上报时可按空间组织。
- 当前空间：只能有一个空间处于 `is_active=true`，控制台顶部显示当前空间。
- 审计：创建、更新、切换、删除空间都会写入审计日志。

## 感知能力边界

空间设置里包含以下规划字段：

- `camera`
- `face_recognition`
- `emotion_recognition`
- `location_tracking`
- `image_retention`
- `privacy_mode`

这些字段当前只表示规划状态：

- `disabled`：关闭。
- `planned`：计划接入，但当前不会采集或处理真实数据。
- `local_only`：后续只允许边缘或本地处理；当前版本仍不会启用真实采集。

当前版本不会采集摄像头画面，不做人脸识别、情绪识别或精确位置追踪。严格隐私模式会强制 `image_retention=none`。

## 后续接树莓派摄像头时的要求

1. 先在 `/spaces` 建立空间和区域。
2. 在 `/devices` 中预建树莓派或摄像头网关设备，默认只读。
3. 设备注册、心跳和遥测仍使用统一 `aiot.v1` 接口。
4. 摄像头、人脸、情绪和定位类数据必须另走高敏感策略评估，不能混入普通环境遥测。
5. 默认不上传原始图像；如需视觉结果，只能先做本地推理，再上传最小化事件摘要。
6. 所有启用、确认、识别结果写入和拒绝都必须有审计日志。
