# 架构说明

个人空间智能物联助手当前版本是一个作品集优先的模拟实现，用来展示个人空间智能体系统的端到端形态。

```text
前端控制台
  -> 后端接口
    -> 确定性模拟遥测
    -> 智能体工具层
    -> 策略引擎
    -> 本地规则与审计日志
```

## 运行模块

- `apps/web`：公开项目页和控制台。
- `services/api/app/routes`：前端使用的后端接口。
- `services/api/app/mock_data.py`：确定性传感器与设备数据。
- `services/api/app/agent_tools.py`：工具优先的智能体编排。
- `services/api/app/policy.py`：风险分级、确认要求和拒绝逻辑。
- `services/api/app/audit.py`：持久化审计记录。

## 当前版本数据流

1. 前端控制台请求当前房间状态或历史趋势。
2. 后端根据时间窗口返回模拟读数。
3. 智能体请求会被映射到受约束工具。
4. 工具返回结构化数据和必要的策略判断。
5. 控制尝试会被允许、要求确认或拒绝。
6. 关键事件持久化到 `services/api/.local/`。

## 下一阶段替换点

- 用消息队列写入的 ESP32 真实遥测替换 `mock_data.py`。
- 用数据库设备注册表替换静态设备清单。
- 用 PostgreSQL 或 TimescaleDB 替换本地 JSON 持久化。
- 保持智能体、策略和审计接口稳定。
